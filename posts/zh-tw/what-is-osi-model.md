# OSI是什麼?

**OSI 模型**是一套全球統一的『網路通訊標準』，它定義了數據從 A 點到 B 點時，這個數據該怎麼描述，讓中間經過的每一台機器知道目地與來源，原始數據會經過一層一層的加工，每一層該穿什麼衣服（格式）、誰負責傳送（接收方）、要去哪裡（目的）、我是誰（來源），以及如何加密（安全）。

### 資料格式的定義
例如 Layer 7 的 HTTP/HTML 或 Layer 4 的 TCP 段 。

### 誰接收的定義（接收方）
例如 Layer 2 透過 MAC 地址 找區網內的設備，Layer 4 透過 Port 找電腦裡的程式 。

### 目的與來源格式
例如 Layer 3 的 IP 地址 標註（來源 IP 與目標 IP） 。

### 加密方式的定義
例如 SSL/TLS 或 SSH 如何在傳輸前保護數據 。

# 封裝（Encapsulation）

Encapsulation就是把Raw Data像是(Hi)，做成010101讓電腦傳遞的過程。📥 封包封裝的「二進位/16進位」演進圖在電腦底層，所有東西最後都會變成 0 與 1（二進位），但工程師為了方便閱讀，會寫成 16 進位 。

**Layer 7:**
原始數據 (Data)內容：你想傳的字串 "A" 。底層實體：在電腦裡 "A" 的 ASCII 碼是 65，轉成 16 進位就是 41。狀態：41

**Layer 4:**
傳輸層標頭 (TCP Header)加入資訊：來源連接埠、目標連接埠（如 443）、序號（Sequence Number）等 。狀態：[TCP 資訊] + 41範例：50 17 01 BB ... 41（前面的數字就是 TCP 的控制資訊）

**Layer 3:**
網路層標頭 (IP Header)加入資訊：來源 IP（你的 IP）與 目標 IP（Google IP）。狀態：[IP 資訊] + [TCP 資訊] + 41範例：45 00 00 3C ... 50 17 01 BB ... 41

**Layer 2:**
資料鏈結層標頭 (Ethernet Frame Header)加入資訊：來源 MAC（你）與 目標 MAC（路由器）。狀態：[MAC 資訊] + [IP 資訊] + [TCP 資訊] + 41最終產物：這就是你在 Wireshark 工具中會看到的那一長串 16 進位代碼 。
